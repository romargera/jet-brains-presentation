## Repo-native loop (viral внутри команды)

1 человек ставит тул, добавляет `/specs` и workflow →
появляются гейты/шаблоны/PR-комменты (в том числе на тему spec drift) →
все, кто трогает репо, вынужденно взаимодействуют со спеками →
другие команды копируют в свои репо.

Ключ: “один репозиторий = один вход, много пользователей”.

## Spec template marketplace loop (контентная петля)

JB и комьюнити делате готовые “spec packs” под типовые задачи (auth, billing, pagination, eventing, migrations) →
команды берут пак, адаптируют →
лучшие версии можно опубликовать обратно (опционально) →
библиотека растет →
новым командам проще стартовать.

Важно: делать это как “пакеты”, а не статьи.

## PR/CI enforcement loop (quality loop)

Внедрили гейты: spec compliance, policy checks, “diff must reference spec” →
меньше багов/ревью-итераций →
команда расширяет покрытие (больше правил, больше компонентов) →
ценность растет с каждой новой проверкой →
больше команд хотят такой же контроль.

Это почти “комплаенс-эффект”: чем больше охват, тем выше switching cost.

## Multi-tool distribution loop (ecosystem loop)

Один и тот же spec-формат работает в Cursor/Claude Code/VS Code/CLI/CI →
команды пробуют “в своем IDE” без миграции →
распространяют внутри орг “как стандарт” →
вы становитесь форматом, а не приложением.

Чем шире совместимость, тем сильнее loop.

## Invite-the-reviewer loop (collaboration loop)

Спек и артефакты упрощают ревью: можно позвать архитектора/безопасника/PM в PR и дать им “читаемый контракт” →
ревью быстрее →
больше людей начинают требовать такой формат →
стандарт закрепляется.

## “Одна спека работает на много репозиториев, и её ценность растет”

Как loop:

Вы создаёте reusable spec/module (например, “billing flows”, “auth”, “error taxonomy”, “event schema”, “API conventions”)

Подключаете к repo A (как пакет/профиль/библиотеку правил)

Дальше тот же артефакт подключается к repo B, C…

Каждое подключение даёт:

больше проверок и совместимости,

меньше расхождений между командами,

накопление “best known spec” в одном месте

Чем больше репо, тем больше стимул:

поддерживать спеку,

расширять её покрытие,

тащить её дальше по организации или в сообщество
