## NSM

* Active consumers/week with >=1 spec-linked merge

## Key metrics

* New MRR (paid stream продажи продукта)
  * Activation to paid
  * ARPU
  * Churn rate (IC / seats)

* Attributed MRR JB core products
  * Attach rate: доля платящих JB, где включен продукт
  * ARPU платящих JB, где включен продукт
  * Churn rate платящих JB, где включен продукт (IC / seats)

* Expantion as %% org with X+ users with >=1 spec-linked merge

## Guardrails

* CR to retention-frinedly core JB actions
* Security/privacy incidents

## Monitoring

### Engagement

* Activation rate to the first spec-linked merge
* N-day retention of users with >=1 spec-linked merge
* Stickiness WAU/MAU, DAU/WAU
* Lead time и deployment frequency

### Basic Quality

* False positive rate у gates/checks
* Latency to PR
* Time to PR
* Token cost per governed change
* Drift rate: доля случаев, где код разошелся со spec (и насколько)
* Drift prevented: сколько drift-инцидентов было поймано до merge

### Value

* Total avg token spend
* доля PR, прошедших проверки с первого раза
